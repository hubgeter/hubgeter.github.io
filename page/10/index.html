虚析构函数

虚析构函数的主要作用是防止内存泄露。

定义一个基类的指针p，在delete p时，如果基类的析构函数是虚函数，这时只会看p所赋值的对象，如果p赋值的对象是派生类的对象，就会调用派生类的析构函数（毫无疑问，在这之前也会先调用基类的构造函数，在调用派生类的构造函数，然后调用派生类的析构函数，基类的析构函数，所谓先构造的后释放）；如果p赋值的对象是基类的对象，就会调用基类的析构函数，这样就不会造成内存泄露。

如果基类的析构函数不是虚函数，在delete p时，调用析构函数时，会进行静态绑定，只会看指针的数据类型，而不会去看赋值的对象，这样就会造成内存泄露。

没有多态，则静态绑定 => call xxx

构造与析构的多态

关于构造函数可以是虚函数的问题，虚函数是通过虚函数表来实现的，而虚函数表是在对象的构造函数中初始化的。也就是说，在对象构造过程中，虚函数表还没有被初始化，因此虚函数的调用不能按照多态的方式进行。

其次，对于析构函数中调用虚函数的问题，虚函数机制在析构函数中依然可以正常工作。由于在对象析构时，其派生类的部分已经被销毁，因此派生类的虚函数将无法被调用。但是基类中的虚函数依然可以被正确地调用。

reserve和resize的区别是什么

reserve只是开辟空间并不创建元素。而resize重新开辟空间并自动初始化元素。

vector的resize函数用于改变vector的大小，如果新的大小比原来的大小大，则在vector的末尾添加新的元素，如果新的大小比原来的大小小，则删除vector末尾的元素。如果新的大小与原来的大小相同，则不进行任何操作。resize函数的返回值为void。

vector的reserve函数用于请求vector预留空间，以便在添加新元素时不必重新分配内存。reserve函数的参数为预留空间的大小，返回值为void。注意，reserve函数只是预留空间，并不会改变vector的大小，如果要改变vector的大小，需要使用resize函数。

push_back 和 emplace_back 的区别

emplace_back的主要优点在与既不会创建也不会析构临时对象。

emplace更快的情况

  1. 预添加的值是以构造而非赋值方式加入容器
  2. 传递的实参型别与容器的持有之物的型别不同
  3. 容器不会由于重复情况而拒绝待添加的新值(欲检测某值是否在容器中，emplace_back会使用新值创建节点，来进行比较，如果该值已存在，置入就会中止，节点也会进行析构)

at()和[]访问的区别

vector的at()方法会进行越界检查，如果越界会抛出out_of_range异常，而[]不会进行越界检查，如果越界会导致未定义行为。

memory order

以这个原子操作为中心，其前后的代码。这些代码并不一定是原子操作

memory order : 编译器以及CPU对单线程当中的指令执行顺序进行重排的程度（此外还包括对cache的控制方法）
这种限制，决定了以atom操作为基准点（边界），对其之前的内存访问命令，以及之后的内存访问命令，能够在多大的范围内自由重排（或者反过来，需要施加多大的保序限制）。从而形成了6种模式。它本身与多线程无关，是限制的单一线程当中指令执行顺序。但是（合理的）指令执行顺序的重排在单线程环境下不会造成逻辑错误而在多线程环境下会，所以这个问题的目的是为了解决多线程环境下出现的问题。

如何理解 C++11 的六种 memory order

memory_order_seq_cst : 编译器和CPU必须严格按照源码逻辑流程的先后顺序。在相同的线程上，以该操作为界，其后的任何操作不得重新编排到它的前面，而前方的任何操作不得重新编排到他的后面。
任何：带有任何内存标记的任何变量之上的任何操作

memory_order_relaxed 在一个线程内，对相同变量的访问次序不能重排
