模板类型推导

expr => T , paramType
paramType 通常会加一些修饰词 例如 const T& , 所以两者通常不一样。

paramType , expr => T

一、paramType 指针/引用，但不是万能引用

  1. expr如果有 引用/指针/对指针的修饰如const ，先忽略
  2. expr 和 paramType 模式匹配 => T

二、ParamType是万能引用

  • expr是左值：T = expr + &

  • expr是右值：与情况一类似

三、ParamType非指针也非引用：按值传递

忽略expr的引用性，const性，volatitle性，来推导T

const

修饰规则

const默认是修饰它左边的符号的，如果左边没有，那么就修饰它右边的符号

用处

const 用于声明常量，就是不可改变的量。
令函数返回常量值，可以降低因客户错误而造成的意外，可以帮助编译器检测出错误用法。
函数传递参数时，const 参数可以接受非const 的实参。

const 常量虽然不是“真正的常数”，但在大多数情况下，它都可以被认为是常数，在运行期间不会改变。编译器看到 const 定义，就会采取一些优化手段，比如把所有 const 常量出现的地方都替换成原始值。
所以，对于没有 volatile 修饰的 const 常量来说，虽然你用指针改了常量的值，但这个值
在运行阶段根本没有用到，因为它在编译阶段就被优化掉了。

volatile ”不稳定的”“易变的” ，禁止编译器做任何形式的优化

mutable 只能修饰类里面的成员变量，表示变量即使是在 const 对象里，也是可以修改的

class const member function

const member functions (常量成员函数) const 不改变数据的时候一定要加

const 与 define

Const , not define .

  • they obey the language’s scoping rules
  • you can see them in the debugger
  • you can take their address if you need to
  • you can pass them by const-reference if you need to
  • they don’t create new “keywords” in your program.
