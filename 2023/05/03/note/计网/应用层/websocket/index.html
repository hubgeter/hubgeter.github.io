一、简介
服务器主动发消息给客户端的场景，解决了轮询造成的同步延迟问题、服务器需要反复解析 HTTP 协议。
对于客户端和服务端之间需要频繁交互的复杂场景，比如网页游戏，都可以考虑使用 WebSocket 协议。
怎么样才能在用户不做任何操作的情况下，网页能收到消息并发生变更: http不断轮询/长轮询 => 本质上还是客户端主动取数据

二、流程
TCP三次握手之后
使用HTTP协议先进行一次通信

以后就用WekSocket协议进行通信
