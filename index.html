对于空类，编译器不会生成任何的成员函数，只会生成1个字节的占位符。

使用空类定义对象或者调用成员函数时，编译器会自动为你生成以下个成员函数（仅在需要时生成）：

  • 一个默认构造函数
  • 一个复制构造函数
  • 一个复制赋值运算符
    copy ctor 和 copy assignment operator 编译器只是单纯的的将source object 的每一个non-static data members 拷贝到 destination object
  • 一个析构函数 （编译器产出的dtor 是non-virtual 除非class 的base class 本身带有virtual dtor）
  • 移动构造函数
  • 移动赋值函数
  • 两个取址运算符（常量和非常量）

这些成员函数都是公有的、内联的、无参数的。
用处：主要是给编译器一个地方用来放置 幕后的code , 例如 base class ,non-static members 的ctors 、dtors

可以使用delete 或者private 来阻止编译器生成

声明复制操作 ，不会生成移动操作

声明移动操作 ，不会生成复制操作

析构函数 阻止移动操作的生成 ，针对移动操作的请求仍然可以被编译和执行，但触发的是复制操作

虚析构函数

虚析构函数的主要作用是防止内存泄露。

定义一个基类的指针p，在delete p时，如果基类的析构函数是虚函数，这时只会看p所赋值的对象，如果p赋值的对象是派生类的对象，就会调用派生类的析构函数（毫无疑问，在这之前也会先调用基类的构造函数，在调用派生类的构造函数，然后调用派生类的析构函数，基类的析构函数，所谓先构造的后释放）；如果p赋值的对象是基类的对象，就会调用基类的析构函数，这样就不会造成内存泄露。

如果基类的析构函数不是虚函数，在delete p时，调用析构函数时，会进行静态绑定，只会看指针的数据类型，而不会去看赋值的对象，这样就会造成内存泄露。

没有多态，则静态绑定 => call xxx

构造与析构的多态

关于构造函数可以是虚函数的问题，虚函数是通过虚函数表来实现的，而虚函数表是在对象的构造函数中初始化的。也就是说，在对象构造过程中，虚函数表还没有被初始化，因此虚函数的调用不能按照多态的方式进行。

其次，对于析构函数中调用虚函数的问题，虚函数机制在析构函数中依然可以正常工作。由于在对象析构时，其派生类的部分已经被销毁，因此派生类的虚函数将无法被调用。但是基类中的虚函数依然可以被正确地调用。
